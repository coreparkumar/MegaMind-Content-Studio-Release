# Publishing Instructions

## Goal

Publish this folder as a separate public repository and create a GitHub release named `Version 1`.

## Repository Contents

- `README.md`
- `USER-INSTRUCTIONS-EN.md`
- `RELEASE_NOTES_V1.md`
- `CHECKSUMS.txt`
- `artifacts/version-1/Megamind Content Studio Setup 1.0.0.exe`
- `artifacts/version-1/Megamind Content Studio Setup 1.0.0.exe.blockmap`

## GitHub Release Steps

1. Create a new public GitHub repository
2. Copy this folder into the new repository root
3. Commit and push the contents
4. Create a release named `Version 1`
5. Upload both files from `artifacts/version-1`

## Release Notes Summary

- Desktop installer built from Version `1.0.0`
- Main workflow supports research generation, review, formatting, and final copy
- This release still requires local backend prerequisites described in `USER-INSTRUCTIONS-EN.md`

## Limitation

This workspace can prepare the repository contents locally, but publishing the public repository and creating the GitHub release still requires your remote Git hosting access.
